# 레포 분리 가이드 — 업무 도우미를 새 레포로

이 레포(`rocketstart`) 하나에 서로 남남인 제품 **3개**가 섞여 있었습니다.

| 제품 | 폴더 | 어디로 |
|------|------|--------|
| **업무 도우미**(학교 공문 로컬 처리) | `src/ app/ docs/ tests/ prototype/` + 루트 `README.md`·`CLAUDE.md`·`requirements*.txt`·`verify_*.py` | **새 레포 `upmu-doumi`** |
| **100시간 공부 앱** | `study/ ios-app/` | `100study` 레포 (다른 작업으로 이전 중) |
| **로켓스타트** | `rocketstart-web-mvp/ rocketstart-docs/` | 이 레포에 그대로 남김 |

> 왜 나누나: 셋은 코드를 한 줄도 공유하지 않고 언어·배포·사용자가 다 다릅니다.
> 레포 하나 = 제품 하나로 두면 AI(Claude)가 엉뚱한 파일을 건드릴 일이 없고,
> 각 제품이 자기 `CLAUDE.md`·배포 설정을 따로 가질 수 있습니다.

---

## 1단계 — GitHub에서 빈 레포 만들기 (클릭 몇 번)

1. github.com 로그인 → 오른쪽 위 **＋ → New repository**
2. **Repository name**: `upmu-doumi`
3. **Private** 선택 (권장)
4. **"Add a README file" 체크 해제** ← 반드시 비워둬야 합니다(빈 레포여야 함)
5. **Create repository** 클릭

---

## 2단계 — 코드 옮기기 (히스토리까지 그대로)

두 방법 중 **하나만** 하면 됩니다. 지금까지의 작업 기록(MVP1~4 커밋)이 모두 보존됩니다.

### 방법 A — 받은 꾸러미 파일 사용 (제일 쉬움)

Claude가 보내준 `upmu-doumi.bundle` 파일을 내 컴퓨터(예: 다운로드 폴더)에 둔 뒤,
터미널에서 그 폴더로 이동해 아래를 붙여넣습니다.

```bash
git clone upmu-doumi.bundle upmu-doumi
cd upmu-doumi
git remote set-url origin https://github.com/dream0grow/upmu-doumi.git
git push -u origin main
```

### 방법 B — 파일 없이, 원본을 복제해서 정리 (꾸러미가 없을 때)

```bash
git clone https://github.com/dream0grow/rocketstart.git upmu-doumi
cd upmu-doumi
git rm -r study ios-app rocketstart-web-mvp rocketstart-docs
git rm .github/workflows/deploy-pages.yml
git commit -m "레포 분리: 업무 도우미만 남김"
git remote set-url origin https://github.com/dream0grow/upmu-doumi.git
git branch -M main
git push -u origin main
```

끝나면 https://github.com/dream0grow/upmu-doumi 에서 `src/ app/ docs/ …` 가 보이는지,
**Commits** 수가 원본과 비슷한지(히스토리 보존) 확인하세요.

---

## 3단계 — 이 레포(rocketstart) 청소하기 (나중에, 선택)

업무 도우미가 새 레포에서 잘 도는 걸 확인한 **뒤에** 진행하세요.
(공부앱 폴더 `study/ ios-app/` 제거는 `100study` 이전 작업에서 처리 중이니 여기서 중복으로 지우지 마세요.)

```bash
# rocketstart 레포에서 (main 말고 새 브랜치 권장)
git rm -r src app docs tests prototype
git rm requirements.txt requirements-dev.txt verify_ai.py verify_demo.py
# 루트 README.md·CLAUDE.md 를 로켓스타트 내용으로 교체
git commit -m "업무 도우미 코드 분리 — 로켓스타트만 유지"
```

정리가 끝나면 이 레포에는 `rocketstart-web-mvp/`·`rocketstart-docs/` 만 남아
레포 이름(`rocketstart`)과 내용이 맞아떨어집니다.

> 3단계는 Claude에게 "rocketstart 청소해줘" 하면 이 브랜치에서 대신 해줄 수 있습니다.
